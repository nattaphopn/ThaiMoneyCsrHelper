Privacy Policy — CSRHelper

Last updated: May 2026

CSRHelper is a private Chrome extension developed for internal use by ThaiExpress customer service representatives.

1. Who This Extension Is For

CSRHelper is available only to authorised ThaiExpress staff. It is published as a private/unlisted extension and is not available to the general public.

2. Data the Extension Accesses

  • Customer display names — Read from LINE OA (chat.line.biz) to pre-fill CRM fields. Not stored by the extension.
  • Authentication token — A session token issued by the internal ThaiExpress service, stored locally in chrome.storage.local on the user's device solely to authenticate API requests.
  • Opportunity and lead data — Entered by the user in the side panel and submitted to the internal ThaiExpress CRM server.
  • Page content on th.admin.eui.money — Read to auto-fill member records. Not stored by the extension.

3. How Data Is Used

  • Display lead status badges on the LINE OA chat list
  • Pre-fill customer information in the CRM side panel
  • Create and update sales opportunities in the internal CRM
  • Authenticate the user's session with the internal API

4. Data Sharing

No data is shared with any third party. All data is transmitted exclusively to internal ThaiExpress servers (service.th.eui.money) over HTTPS. No analytics, advertising, or tracking services are used.

5. Data Storage

  • Authentication token is stored locally in chrome.storage.local on the user's device
  • Lead and opportunity data is stored on the internal CRM server under ThaiExpress's data retention policies
  • No cookies or third-party cloud storage are used

6. Permissions Used

  • activeTab / scripting — Read customer name from LINE OA tab on user action only
  • contextMenus — Right-click shortcut to open the CRM panel
  • sidePanel — Display the CRM interface alongside LINE OA
  • storage — Persist the session token locally
  • tabs — Detect chat navigation to keep the side panel in sync

7. Security

All communication uses HTTPS. The authentication token is stored only on the local device and never transmitted to any party other than the authorised internal API.
